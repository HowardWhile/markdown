# 為什麼工程師都愛用 Markdown 寫文件？

**嗨，大家！你是否曾經遇過這些問題？**

1. **排版地獄**：在 Word 裡調整圖片位置，結果整份文件的文字都跑掉了。
2. **格式不相容**：把文件內容複製到 Email 或 LINE，格式瞬間亂成一團。
3. **專注力分散**：寫報告時，花在「調整字體大小、行距」的時間，比「寫內容」的時間還多。

如果你有同感，那麼你一定要認識這個提升效率的神器  **Markdown**。



# 什麼是 Markdown？

Markdown 不是程式語言，它是一種 **「輕量級的標記語法」**。 簡單來說，你只需要用鍵盤打出幾個簡單的符號（如 `#` 或 `-`），就能自動產生排版好的文件。

**它的核心理念是：雙手不離鍵盤，專注於「寫作內容」，而非「滑鼠排版」。**

目前我們的 **版控系統**、github、Slack、Notion、甚至 ChatGPT，全都支援這種語法！



# 只要記住這 5 招，你就能上手

不用死記硬背，Markdown 的邏輯非常直覺：

## 1. 標題 (Headings)

想設標題？在文字前面加 `#` 就好。

- `# 大標題` (對應 Word 的標題 1)
- `## 中標題` (對應 Word 的標題 2)
- `### 小標題`

## 2. 列表 (Lists)

想列清單？用減號 `-` 或數字 `1.` 開頭。

- `- 項目 A`
- `1. 第一點`

## 3. 強調 (Emphasis)

想畫重點？用星號包起來。

- `**這是粗體**`
- `*這是斜體*`

## 4. 待辦事項 (Checklist) 

在開會記錄或任務分配時超好用！

- `- [ ] 尚未完成的工作`
- `- [x] 已完成的工作`

## 5. 插入程式碼或強調文字

如果文字中有英文指令或不想被翻譯的內容，用反引號 `     ` ` 包起來。

- 請輸入 `login` 指令

### 1. 標題與文字格式

最基礎的排版功能。

Markdown

```
# 一級標題 (通常用於文件大標)
## 二級標題 (章節)
### 三級標題 (小節)

**這是粗體文字**
*這是斜體文字*
~~這是刪除線~~
`這是行內程式碼 (Inline Code)`
```

------

### 2. 程式碼區塊 (Syntax Highlighting)

這是工程師最常用的。Gitea 支援多種語言的高亮顯示 (Python, C++, Bash, JSON, YAML 等)。

**一般程式碼：**

Markdown

````
```python
# 這是 Python 範例
def hello():
    print("Hello Gitea")
```
````

**Diff 語法 (顯示修改前後)：** 這在寫 Release Note 或教學時非常有用。

```diff
  function test() {
-   console.log("舊的程式碼");
+   console.log("新的修正");
  }
```

````
```diff
  function test() {
-   console.log("舊的程式碼");
+   console.log("新的修正");
  }
```
````

### 3. 待辦清單 (Task Lists)

在 Gitea 的 Issue 或 PR 中，這種語法會變成可勾選的互動框框。

- [x] 已完成的工作項目
- [ ] 尚未開始的項目
- [ ] 進行中的項目

```
- [x] 已完成的工作項目
- [ ] 尚未開始的項目
- [ ] 進行中的項目
```

### 4. 表格 (Tables)

用來整理參數或數據。冒號的位置決定對齊方向。

| 欄位 A | 欄位 B (置中) | 欄位 C (靠右) |
| :----- | :-----------: | ------------: |
| 內容 1 |     數值      |           100 |
| 內容 2 |     測試      |           200 |

```
| 欄位 A | 欄位 B (置中) | 欄位 C (靠右) |
| :--- | :---: | ---: |
| 內容 1 | 數值 | 100 |
| 內容 2 | 測試 | 200 |
```

------

### 5. 引用與區塊 (Blockquotes)

用來強調筆記或警告。

Markdown

```
> 這是一段引用文字
> 這是第二行
>
> > 這是巢狀引用 (引用中的引用)
```

------

### 6. 連結與圖片

Markdown


### 7. 折疊區塊 (Details)

如果 Log 太長或內容太細節，可以用這個把它收合起來，讓版面乾淨。

Markdown

```
<details>
<summary>點擊這裡展開查看詳細 Log</summary>

這裡是被隱藏的內容...
1. Error log line 1
2. Error log line 2
可以放很長很長的文字。

</details>
```
